# Component Naming

Kinetics uses functional component names.

Names describe the user-facing role or behavior:

- `ActionControl`
- `TextEntry`
- `ChoiceMark`
- `StateSwitch`
- `ViewSwitcher`
- `ActionBar`
- `NavigationRail`
- `MetricReadout`
- `BlankState`
- `ModalLayer`
- `NoticeStack`
- `CommandFinder`
- `ContextHint`
- `ContentPlane`
- `GlassLayer`
- `TimelineScope`
- `FrameStage`
- `CaptureStage`
- `TrendLine` (`Sparkline`)
- `TrendChart` (`LineChart`)
- `ComparisonChart` (`BarChart`)
- `ProgressDial` (`DonutGauge`)
- `ReorderList` (`SortableList`)
- `WorkflowBoard` (`KanbanBoard`)
- `GuidedTour` (`Tour`)
- `AudioLevels` (`Waveform`)

Public names do not borrow library, framework, platform, animation, or video product names.
The previous MVP names can remain available during the `0.1.x` transition, but the default
documentation and examples should prefer the functional names.
